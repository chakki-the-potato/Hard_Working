## Notion을 제대로 쓰는 핵심: 데이터베이스

단순히 텍스트를 적는 메모장으로 쓰면 Notion의 10%도 못 씁니다. 핵심은 **데이터베이스**입니다.

## 개발자용 Notion 구조

```
📁 개발 공부
  ├── 📊 학습 로그 (Database)
  ├── 📚 북마크 (Database)
  ├── 🐛 문제 해결 노트 (Database)
  └── 📝 프로젝트 (Database)
```

## 학습 로그 데이터베이스 세팅

| 속성 | 타입 | 용도 |
|------|------|------|
| 제목 | Title | 오늘 공부한 주제 |
| 날짜 | Date | 학습일 |
| 태그 | Multi-select | JavaScript, React 등 |
| 상태 | Select | 완료/진행중/복습필요 |
| 시간 | Number | 학습 시간(분) |
| 링크 | URL | 참고 자료 |

## 유용한 뷰 설정

- **캘린더 뷰**: 날짜별 학습 현황 한눈에
- **갤러리 뷰**: 커버 이미지로 시각적 정리
- **보드 뷰**: 상태별(완료/진행중) 칸반 보드
- **타임라인**: 프로젝트 일정 관리

## 템플릿 활용

매일 쓰는 학습 노트 템플릿:

```
# {{날짜}} 학습 노트

## 오늘 배운 것
-

## 코드 스니펫
```코드```

## 모르는 것 / 더 공부할 것
-

## 내일 할 것
-
```

## Notion AI 활용법

- 복잡한 개념 요약 요청
- 코드 설명 요청
- 학습 계획 생성
- 영어 문서 번역

## 북마크 데이터베이스

웹에서 좋은 글을 발견했을 때 즉시 저장하는 시스템.

속성: 제목, URL, 태그(분야), 읽음여부, 저장날짜

Notion Web Clipper 브라우저 확장을 설치하면 원클릭으로 저장 가능합니다.
