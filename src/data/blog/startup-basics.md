## 왜 대부분의 스타트업은 실패하는가

CB Insights의 분석에 따르면 스타트업 실패의 1위 원인은 **"시장이 원하지 않는 제품"** 을 만들었기 때문입니다. 기술이 부족해서도, 팀이 나빠서도 아닙니다.

> 빠르게 만들고, 빠르게 검증하고, 빠르게 버려라.

## 린 스타트업 핵심 루프

```
아이디어 → Build → 제품 → Measure → 데이터 → Learn → 아이디어
```

핵심은 **Build-Measure-Learn** 사이클을 최대한 빠르게 돌리는 것입니다.

## MVP란 무엇인가

MVP(Minimum Viable Product)는 "최소한의 기능으로 검증 가능한 제품"입니다.

흔한 오해: MVP = 기능이 적은 완성품
실제 의미: MVP = 가설을 검증하기 위한 가장 빠른 수단

### MVP 예시

| 가설 | MVP 방법 |
|------|---------|
| 배달 앱 수요 있음 | 전화 주문 + 직접 배달로 테스트 |
| 구독 서비스 원함 | 랜딩 페이지 + 사전 결제 |
| 커뮤니티 원함 | 오픈카카오톡방 운영 |

## PMF(Product-Market Fit) 찾기

숀 엘리스 테스트: **"이 제품이 없어진다면 얼마나 실망하겠습니까?"**
- 매우 실망: 40% 이상 → PMF 달성 신호
- 40% 미만 → 아직 멀었다

### PMF를 찾았다는 신호
- 입소문이 자연스럽게 퍼진다
- 고객이 먼저 기능을 요청한다
- 이탈률이 낮고 재방문율이 높다
- 팀이 공급을 따라가지 못할 정도로 수요가 있다

## 팀 구성의 원칙

초기 스타트업에 필요한 세 가지 역할:

1. **Hacker** — 제품을 만드는 사람
2. **Hustler** — 영업/마케팅을 하는 사람
3. **Designer** — 사용자 경험을 설계하는 사람

혼자 하는 창업은 가능하지만, 두세 명이 시작하는 것이 훨씬 유리합니다.

## 초기에 하지 말아야 할 것들

- 법인 설립을 너무 빨리 하기
- 사무실 임대
- 채용을 서두르기
- 투자 유치에만 집중하기
- 경쟁사 분석에 시간 쏟기

## 첫 고객을 어떻게 얻는가

Paul Graham의 조언: **"Do things that don't scale"**

- 직접 전화하기
- 커뮤니티에서 발로 뛰기
- 지인 네트워크 활용
- SNS DM 보내기

규모화는 나중에 해도 됩니다. 지금은 첫 10명의 팬을 만드는 게 먼저입니다.
